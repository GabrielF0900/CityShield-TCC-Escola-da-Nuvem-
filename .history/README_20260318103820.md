# CityShield: Segurança Integrada Inteligente para Cidades Inteligentes

## 📋 Sobre Este Projeto

**CityShield** é uma proposta de plataforma unificada de segurança desenvolvida como Trabalho de Conclusão de Curso (TCC) do programa **AWS Re/Start + IA** ministrado pela [Escola da Nuvem](https://escoladanuvem.org).

A inovação central do CityShield é a convergência de dois domínios tradicionalmente isolados: **Segurança Física** e **Segurança Cibernética**, em uma única "central de inteligência" baseada em nuvem, capaz de tomar decisões automáticas em tempo real.

---

## 🏗️ Visão Geral da Arquitetura

### Paradigma: Event-Driven Architecture (Arquitetura Orientada por Eventos)

A arquitetura do CityShield é **baseada em eventos** e utiliza **serviços serverless** da AWS para garantir:
- 📈 **Escala infinita**: Cidades geram volumes massivos de dados
- 💰 **Otimização de custo**: Pagamento apenas pelo uso efetivo dos recursos
- ⚡ **Latência reduzida**: Processamento em tempo real

### Convergência de Dois Fluxos Críticos

O CityShield processa dois fluxos de segurança que convergem em uma resposta unificada:

```
┌─────────────────────────┐
│ Fluxo 1: Segurança      │
│ Pública Física          │  ──────┐
│ (O "Olho" da Cidade)    │        │
└─────────────────────────┘        │
                                   ├──→ [Central Unificada] ──→ [Resposta Automática]
┌─────────────────────────┐        │
│ Fluxo 2: Segurança      │        │
│ Cibernética             │  ──────┘
│ (Sistema Imunológico)   │
└─────────────────────────┘
```

## 🎥 Fluxo 1: Segurança Pública Física

### Pipeline de Detecção Facial

#### 1️⃣ **Fonte**: Câmeras Distribuídas
- **Componente**: Câmeras de segurança espalhadas pela cidade
- **Função**: Captura contínua de vídeo em tempo real

#### 2️⃣ **Ingestão de Vídeo**: Amazon Kinesis Video Streams
- **Função**: Receptor de streaming de vídeo robusto
- **Capacidade**: 
  - Processa fluxos de milhares de câmeras simultaneamente
  - Ingestão segura sem ponto de gargalo
  - Funciona como um "tubo gigante" capaz de receber dados contínuos

#### 3️⃣ **Processamento de IA Visual**: Amazon Rekognition
- **Função**: Análise inteligente de imagem baseada em machine learning
- **Ações configuradas**:
  - Detecção e extração de faces em cada frame
  - Análise de confiança (high confidence matches)

#### 4️⃣ **Comparação com Banco de Dados**: Coleção Rekognition
- **Base de dados**: Coleção pré-carregada de faces de procurados
- **Processo**: 
  - Cada face detectada é comparada contra a coleção
  - Quando há "match" com alta confiança, um evento é disparado

#### 5️⃣ **Orquestração**: AWS Lambda
- **Função**: Lógica sem servidor acionada por evento
- **Entrada**: Alerta "Procurado X detectado na Câmera Y às Z horas"
- **Processamento**: Estruturação e formatação do incidente

#### 6️⃣ **Notificação Crítica**: Amazon SNS (Simple Notification Service)
- **Canais de distribuição**:
  - 📱 **SMS**: Comandante da polícia da área afetada
  - 📞 **Sistema de Despacho**: SAMU/Bombeiros (conforme tipo de incidente)
  - 🎯 **Sistema de Resposta**: Centros integrados de comando

## 🔐 Fluxo 2: Segurança Cibernética

### Pipeline de Detecção de Ameaças

#### 1️⃣ **Fonte**: Infraestrutura Digital da Cidade
- **Componentes**:
  - 🖥️ Servidores da Prefeitura
  - 📡 Dispositivos IoT (semáforos inteligentes, sensores urbanos)
  - 🌐 Portais de serviços ao cidadão

#### 2️⃣ **Monitoramento Contínuo**: Amazon GuardDuty
- **Função**: Serviço de detecção automática de ameaças com machine learning
- **Fontes analisadas**:
  - **VPC Flow Logs**: Tráfego de rede da cidade
  - **CloudTrail**: Auditoria de APIs e ações na AWS
  - **DNS Logs**: Resolução de domínio (detecção de C&C, malware)
- **Inteligência**:
  - Aprende padrões normais de comportamento
  - Identifica anomalias características de ataques (ex: força bruta em SSH, exfiltração de dados)
- **Exemplos de detecção**:
  - ❌ Um IP tentando milhares de senhas diferentes (força bruta)
  - ❌ Um servidor da prefeitura tentando comunicação com IP conhecido por malware
  - ❌ Padrões de exfiltração de dados

#### 3️⃣ **Captura de Evento**: Amazon EventBridge
- **Função**: Roteador de eventos inteligente
- **Ação**: Captura alertas de segurança de alto risco emitidos pelo GuardDuty

#### 4️⃣ **Resposta Automática**: AWS Lambda (Cibernético)
- **Função**: Orquestrador que toma ações imediatas
- **Integração**: Comunica-se com AWS WAF para bloqueio instantâneo

#### 5️⃣ **Bloqueio em Tempo Real**: AWS WAF (Web Application Firewall)
- **Ação automática**: 
  - Adiciona IPs hostis a listas de bloqueio instantaneamente
  - Mitiga DDoS, força bruta e ataques de camada 7
- **Tempo de resposta**: Segundos (sem intervenção humana)

## 🔄 Convergência e Auditoria

### Consolidação Centralizada

#### 📊 **DynamoDB**: Registro Centralizado
- **Função**: Banco de dados NoSQL para armazenamento de incidentes em tempo real
- **Dados armazenados**:
  - Incidentes de segurança física (detecções faciais)
  - Incidentes de segurança cibernética (ameaças bloqueadas)
- **Benefício**: Painel de controle único com histórico completo

#### 🔍 **Amazon S3**: Armazenamento Forense
- **Tipos de evidência**:
  - 🎬 Vídeo raw dos incidentes (segurança física)
  - 📋 Logs brutos dos ataques (segurança cibernética)
- **Políticas**:
  - Retenção configurada por conformidade legal
  - Indexação para busca rápida em investigações
  - Integração com ferramentas de análise forense

## 🛠️ Tecnologias Utilizadas

### Serviços AWS (Serverless)

| Serviço | Categoria | Função |
|---------|-----------|--------|
| **Kinesis Video Streams** | Ingestão | Streaming de vídeo de múltiplas câmeras |
| **Rekognition** | IA/ML | Detecção e comparação facial |
| **Lambda** | Orquestração | Lógica sem servidor, acionada por eventos |
| **SNS** | Notificações | Distribuição de alertas (SMS, push, webhook) |
| **GuardDuty** | Segurança | Detecção de ameaças cibernéticas com ML |
| **EventBridge** | Eventos | Roteamento e filtragem de eventos |
| **WAF** | Proteção | Bloqueio em tempo real de IPs/ataques |
| **DynamoDB** | Banco de Dados | Armazenamento de incidentes (tempo real) |
| **S3** | Armazenamento | Evidências e logs (conformidade legal) |

### Paradigmas e Práticas

- 🏗️ **Event-Driven Architecture**: Baseado em eventos para reatividade máxima
- 📈 **Serverless Computing**: Escalabilidade automática e opex otimizado
- 🤖 **Machine Learning**: Detecção inteligente sem regras hardcoded
- 🔗 **Integração em Tempo Real**: Fluxos que convergem para resposta unificada
